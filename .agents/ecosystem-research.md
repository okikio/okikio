# Ecosystem research

Use this note when a task needs current framework or platform guidance.

Prioritize:
- Astro documentation for `.astro` component, routing, and build behavior
- Tailwind CSS guidance for styling patterns
- accessibility guidance for navigation, imagery, focus states, and responsive content
- platform-specific docs when touching adapters, Cloudinary usage, or deployment config

Research should answer:
- what the current best practice is
- whether the repo already follows it
- whether a change would improve correctness, accessibility, maintainability, or user experience

Do not copy external advice blindly.
Translate it into the smallest repo-appropriate change.
